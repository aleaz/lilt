# Support

## Questions vs bugs

| Kind | Where |
|------|--------|
| **Bug** (crash, wrong behavior, regression) | [GitHub Issues](https://github.com/aleaz/lilt/issues) — use the Bug Report template |
| **Feature request** | [GitHub Issues](https://github.com/aleaz/lilt/issues) — use the Feature Request template |
| **How do I…?** / product questions | [FAQ](docs/faq.md) and [docs hub](docs/README.md); then open an issue labeled as a question if still stuck |
| **Something broke / how do I recover?** | [Troubleshooting](docs/runbooks/troubleshooting.md) → [Recovery](docs/runbooks/recovery.md) → [Error reference](docs/runbooks/error-reference.md) |
| **Security vulnerability** | **Do not** open a public issue — see [SECURITY.md](SECURITY.md) |

## Before opening an issue

1. Check [FAQ](docs/faq.md) (conceptual) and [Troubleshooting](docs/runbooks/troubleshooting.md) (symptoms).
2. Try the matching [Recovery](docs/runbooks/recovery.md) guide if applicable.
3. Confirm your install (`lilt --version`) and Python 3.13+.
4. Include OS, command line, and relevant logs (redact API keys).

## Maintainer response

This is a solo-maintained public beta. Response times vary; security reports are prioritized per [SECURITY.md](SECURITY.md).
